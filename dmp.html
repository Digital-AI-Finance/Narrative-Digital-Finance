1 Data collection and documentation

1.1 What data will you collect, observe, generate or reuse?

We use publicly available data from financial markets (accessed via original source URLs) and commercial databases (accessed via provider APIs). Public data is not re-stored by us. Commercial data remains on provider infrastructure per license agreements. Only project-generated code and outputs are stored on GitHub.

Commercial Data Sources (Licensed):

RavenPack: News headlines with sentiment scores. Format: CSV, Parquet. Volume: ~3 GB. Time range: 2000-2025.

Deutsche Borse T7: Nanosecond-level trading data for FESX and DAX futures. Format: Binary, CSV. Volume: ~50 GB. Time range: January-August 2025.

Public Data Sources:

BIS Gigando: Central bank speeches worldwide. Format: Text, PDF. URL: www.bis.org/cbspeeches/

St. Louis FED FRED: Macroeconomic indicators (CPI, PPI, GDP, etc.). Format: CSV via API. URL: fred.stlouisfed.org/

Generated Data:

Project code: Analysis scripts and methods. Format: Python. Volume: minimal. Repository: Zenodo.

1.2 How will the data be collected, observed or generated?

The quality of the collected data will be checked during the first working package and is an integral part of the research. The consistency as well. In addition, due to shortcomings of the data, we will apply statistical methods to overcome this. We will have various documents describing the dataset, its quality and the methods used to check its consistency.

Quality assurance: The data are checked for consistency and completeness using statistical methods.

Versioning: The code and the database are versioned with the help of Git. Code repositories are maintained on GitHub (Digital-AI-Finance organization). Data releases may receive DOIs via Zenodo integration.

Data Collection Methods: API Access via FRED API (Python fredapi) and BIS API. Commercial data is accessed via RavenPack SQL access and Deutsche Borse secure file transfer. NLP Processing uses HuggingFace transformers, BERT/FinBERT embeddings, and BERTopic modeling.

1.3 What documentation and metadata will you provide with the data?

The information on the data as well as data sources and survey processes are documented in detail.

Metadata Standards: Standard README documentation with variable descriptions and usage instructions.

Documentation Provided: Data Dictionaries contain variable names, types, and descriptions in README.md files. Code includes inline comments where helpful. Dependency files (requirements.txt) are provided where applicable.

2 Ethics, legal and security issues

2.1 How will ethical issues be addressed and handled?

No personal data or other sensitive data is used in the project. In this respect, the university internal security standards are applied. The conditions have already been discussed with the data providers. No special security standards are required by the data providers for this data.

Ethical Considerations: All financial data is aggregated market data with no individual identification. Central bank speeches are official public communications. Research outputs are intended for academic purposes.

Data Provider Agreements: RavenPack operates under research subscription with academic use only. Deutsche Borse operates under research collaboration agreement with project-specific usage.

Ethics Approval: Not required (non-human-subjects research per Swiss SNSF guidelines)

2.2 How will data access and security be managed?

We use private cloud solutions. No sensitive data or personal data is collected in the project.

Access Control: Public data (FRED, BIS, SEC) has open access and is stored on Original source URLs (not re-stored by us). Commercial data has provider access only access and is stored on Provider cloud infrastructure (not redistributable). Project code has open access and is stored on GitHub.

Security Measures: Commercial data remains on provider infrastructure per license agreements. Relevant project code is public on GitHub and Zenodo.

2.3 How will you handle copyright and Intellectual Property Rights issues?

The project is based on data that are largely publicly available. The raw data records may not be published without restriction.

Copyright Framework: RavenPack news is copyrighted by RavenPack Inc. with academic use only rights under Proprietary. LSEG transcripts is copyrighted by LSEG with academic use only rights under Proprietary. Deutsche Borse data is copyrighted by Deutsche Borse AG with research collaboration rights under Agreement. BIS speeches is copyrighted by BIS/Speakers with full reuse rights under Public (cite). SEC filings is copyrighted by Public domain with full reuse rights under Public domain. Our code is copyrighted by Project team with open source rights under MIT License. Our models/data is copyrighted by Project team with open access rights under CC-BY 4.0.

Publication Rights: Yes Aggregated statistics and derived features. Yes Code, models, and methodology. No Raw commercial data redistribution.

3 Data storage and preservation

3.1 How will your data be stored and backed-up during the research?

The storage capacities are very large, the amount of data remains very limited in the project. Relevant data is managed in the form of databases on GitHub and Zenodo and private cloud solutions. Backups and versions of the data are continuously created.

Primary Storage: Location is GitHub repositories. Capacity is Unlimited (code only). Backup schedule is git version control. Retention is as long as github available.

3.2 What is your data preservation plan?

Most relevant data is stored on Zenodo.

4 Data sharing and reuse

4.1 How and where will the data be shared?

The relevant code developed during the project together with all necessary accompanying documentation will be stored on Zenodo. Zenodo offers safe storage for all data and research outputs in CERN's Data Centre and it provides easy integration with GitHub.

Current Outputs (Available Now): Project Website is live on GitHub Pages at digital-ai-finance.github.io/Narrative-Digital-Finance/.

Sharing by Data Type: Public data (FRED, BIS, SEC): Yes (Available at original sources - we do not re-distribute). Commercial data (RavenPack, LSEG, Deutsche Borse): No (License restrictions - remains on provider servers). Project code: Yes (GitHub (MIT License)).

4.2 Are there any necessary limitations to protect sensitive data?

We do not use sensitive data in the project. The data come from conventional data providers and are originally collected from public sources.

4.3 All digital repositories I will choose are conform to the FAIR Data Principles.

Yes

FAIR Data Principles Implementation: Findable: Code on GitHub (public repository). Accessible: GitHub with standard HTTPS access. Interoperable: Python code with standard libraries. Reusable: MIT License, README files.

4.4 I will choose digital repositories maintained by a non-profit organisation.

Yes
