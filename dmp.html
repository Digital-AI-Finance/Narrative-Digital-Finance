1 Data collection and documentation

1.1 What data will you collect, observe, generate or reuse?

We will use publicly available data on financial markets as well as data from databases which we buy. The data will be in various formats including CSV, JSON, and Parquet, totaling approximately 65 GB. The data types are financial data, stored as integers, floats, and characters.

Commercial Data Sources (Licensed):

Refinitiv/LSEG: Earnings call transcripts, financial news, market data. Format: JSON, CSV. Volume: ~5 GB. Time range: 2010-2025.

RavenPack: News headlines with sentiment scores. Format: CSV, Parquet. Volume: ~3 GB. Time range: 2000-2025.

Deutsche Borse T7: Nanosecond-level trading data for FESX and DAX futures. Format: Binary, CSV. Volume: ~50 GB. Time range: January-August 2025.

Public Data Sources:

BIS Gigando: Central bank speeches worldwide. Format: Text, PDF. URL: www.bis.org/cbspeeches/

SEC EDGAR: 10-K and 10-Q regulatory filings. Format: HTML, XBRL. URL: www.sec.gov/edgar

St. Louis FED FRED: Macroeconomic indicators (CPI, PPI, GDP, etc.). Format: CSV via API. URL: fred.stlouisfed.org/

Generated Data:

TOPol Embeddings: Transformer-based semantic embeddings. Format: NumPy, HDF5. Volume: ~2 GB. Repository: osf.io/nr94j/.

NLP Features: Sentiment scores, topic models, embeddings. Format: CSV, Parquet. Volume: ~1 GB. Repository: Zenodo (pending).

Macro Index: PCA-based US Macro Strength Index. Format: CSV. Volume: less than 100 MB. Repository: GitHub.

Total estimated data volume: ~65 GB

1.2 How will the data be collected, observed or generated?

The quality of the collected data will be checked during the first working package and is an integral part of the research. The consistency as well. In addition, due to shortcomings of the data, we will apply statistical methods to overcome this. We will have various documents describing the dataset, its quality and the methods used to check its consistency.

Quality assurance: The data are checked for consistency and completeness using statistical methods. Automated validation scripts verify data integrity. Cross-validation with alternative data sources is performed where available.

Versioning: The code and the database are versioned with the help of Git. Code repositories are maintained on GitHub (Digital-AI-Finance organization). Data releases may receive DOIs via Zenodo integration.

Data Collection Methods: API Access via FRED API (Python fredapi), SEC EDGAR XBRL API, and BIS web scraping (Beautiful Soup). Commercial data is accessed via RavenPack SQL access, LSEG API and bulk delivery, and Deutsche Borse secure file transfer. NLP Processing uses HuggingFace transformers, BERT/FinBERT embeddings, and BERTopic modeling.

1.3 What documentation and metadata will you provide with the data?

The information on the data as well as data sources and survey processes are documented in detail. The information on the project and the data will be made available to our university employees so that further projects can be developed in this area.

Metadata Standards: Standard README documentation with variable descriptions and usage instructions.

Documentation Provided: Data Dictionaries contain variable names, types, and descriptions in README.md files. Code includes inline comments where helpful. Dependency files (requirements.txt) are provided where applicable.

2 Ethics, legal and security issues

2.1 How will ethical issues be addressed and handled?

No personal data or other sensitive data is used in the project. In this respect, the university internal security standards are applied. The conditions have already been discussed with the data providers. No special security standards are required by the data providers for this data.

Ethical Considerations: All financial data is aggregated market data with no individual identification. Central bank speeches are official public communications. Research outputs are intended for academic purposes.

Data Provider Agreements: Refinitiv/LSEG operates under academic research license with no redistribution of raw data. RavenPack operates under research subscription with academic use only. Deutsche Borse operates under research collaboration agreement with dr. schlamp with project-specific usage.

Ethics Approval: Not required (non-human-subjects research per Swiss SNSF guidelines)

2.2 How will data access and security be managed?

Access to the data is only granted to team members. The university IT service guarantees the security of data and processes. No sensitive data or personal data is collected in the project.

Access Control: Public data (FRED, BIS, SEC) has open access and is stored on GitHub, Zenodo. Commercial data has team only access and is stored on BFH secure server. Deutsche Borse HFT data has restricted access and is stored on Encrypted local storage. Processed features has open (with publication) access and is stored on OSF, Zenodo.

Security Measures: BFH IT infrastructure with standard institutional security measures. Encrypted file transfer for commercial data where required.

Authorized Team Members: Prof. Dr. Joerg Osterrieder (Principal Investigator), Gabin Taibi (PhD Researcher), Lennart John Baals (PhD Researcher), Yiting Liu (PhD Researcher), Marius Jan Klein (Researcher).

2.3 How will you handle copyright and Intellectual Property Rights issues?

The project is based on data that are largely publicly available. The raw data records may not be published without restriction.

Copyright Framework: RavenPack news is copyrighted by RavenPack Inc. with academic use only rights under Proprietary. LSEG transcripts is copyrighted by LSEG with academic use only rights under Proprietary. Deutsche Borse data is copyrighted by Deutsche Borse AG with research collaboration rights under Agreement. BIS speeches is copyrighted by BIS/Speakers with full reuse rights under Public (cite). SEC filings is copyrighted by Public domain with full reuse rights under Public domain. Our code is copyrighted by Project team with open source rights under MIT License. Our models/data is copyrighted by Project team with open access rights under CC-BY 4.0.

Publication Rights: Yes Aggregated statistics and derived features. Yes Code, models, and methodology. No Raw commercial data redistribution.

3 Data storage and preservation

3.1 How will your data be stored and backed-up during the research?

The storage capacities are very large, the amount of data remains very limited in the project. The data is managed in the form of databases on GitHub. Backups and versions of the data are continuously created.

Primary Storage: Location is BFH servers. Capacity is Sufficient storage allocated. Backup schedule is regular backups per bfh it policy. Retention is as long as technically feasible.

Cloud Storage: GitHub (github.com/Digital-AI-Finance) is used for code repositories. OSF (osf.io/nr94j/) is used for research materials and preprints. Zenodo is used for archived releases with dois, Linked via GitHub.

Backup Strategy: Code is backed up continuous (git) with as long as github available retention on GitHub. Processed data is backed up periodic with as long as feasible retention on BFH + Zenodo. Commercial raw data is backed up per bfh policy with project duration retention on BFH.

3.2 What is your data preservation plan?

The data is stored on GitHub for a long time and managed using existing tools. There is no obligation to destroy the data.

Repository Selection: Zenodo is operated by CERN/OpenAIRE (non-profit), offers DOI assignment, long-term storage. OSF is operated by Center for Open Science (non-profit), offers Research materials. arXiv is operated by Cornell University (non-profit), offers Preprints.

Preservation Timeline: At project end (august 2026), we archive code and shareable outputs with DOI via Zenodo if applicable.

Archival File Formats (Open Standards): Data in CSV, Parquet, HDF5 (machine-readable, open formats). Documentation in Markdown, PDF/A (long-term readable). Code in Python (plain text, version-controlled). Images in PNG, SVG (lossless, open formats).

Retention Period: As long as technically feasible

4 Data sharing and reuse

4.1 How and where will the data be shared?

The code developed during the project together with all necessary accompanying documentation will be stored on a GitHub channel. On the other hand, the data archiving will be done through Zenodo. Zenodo offers safe storage for all data and research outputs in CERN's Data Centre and it provides easy integration with GitHub.

Current Outputs (Available Now): TOPol Framework is published on OSF at osf.io/nr94j/. Project Website is live on GitHub Pages at digital-ai-finance.github.io/Narrative-Digital-Finance/.

Upcoming Deposits: Code and derived outputs will be deposited on GitHub + Zenodo Subject to publication. Additional materials will be deposited on Zenodo or OSF If applicable.

Sharing by Data Type: FRED, BIS, SEC: Yes (Public domain / open access). RavenPack raw data: No (License restriction). LSEG transcripts: No (License restriction). Deutsche Borse T7: No (Collaboration agreement). Derived features: Yes (Our work product). All code: Yes (MIT License).

4.2 Are there any necessary limitations to protect sensitive data?

We do not use sensitive data in the project. The data come from conventional data providers and are originally collected from public sources.

Unrestricted (Open Access): All code developed by the project (MIT License). Derived features, embeddings, and processed outputs (CC-BY 4.0). Documentation, methodology, and research materials. Public source data (FRED, BIS, SEC).

Restricted (Cannot Share): Raw RavenPack news data (license prohibits redistribution). Raw LSEG/Refinitiv transcripts (license prohibits redistribution). Raw Deutsche Borse T7 data (collaboration agreement restriction).

Reuse Conditions: Citation required (BibTeX provided in repositories). License compliance: MIT for code, CC-BY 4.0 for data/models. Academic and commercial use permitted for our outputs.

Access Requests: For restricted data, contact joerg.osterrieder@bfh.ch - we can facilitate contact with original data providers.

4.3 All digital repositories I will choose are conform to the FAIR Data Principles.

Yes

FAIR Data Principles Implementation: Findable: DOIs via Zenodo where applicable. Accessible: Open repositories (Zenodo, OSF, GitHub) with standard protocols. Interoperable: Standard formats (CSV, JSON, Parquet). Reusable: CC-BY license for project outputs, README files.

4.4 I will choose digital repositories maintained by a non-profit organisation.

Yes

Repository Certification: Zenodo is operated by CERN/OpenAIRE, Non-profit, CoreTrustSeal certified. OSF is operated by Center for Open Science, Non-profit, COS Registered Reports partner. arXiv is operated by Cornell University, Non-profit, permanent archive.
